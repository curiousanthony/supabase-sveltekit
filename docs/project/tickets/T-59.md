Now that `prixConvenu`, `prerequis`, `publicVise` are surfaced on the Fiche (T-54), wire them into the document preflight so generating a convention or programme catches the gaps before they cause a Qualiopi finding. `prixConvenu` is **legally required** in the convention (Article L6353-1 Code du Travail) → block. The pédagogique checks → warn (don't stop the world; Marie may want to generate a draft).

Note: formation-level `objectifs` is NOT used. The Fiche shows a read-only roll-up of `modules[].objectifs` (per T-54). Preflight checks the same module-level aggregation: warn when **no module** has `objectifs` defined; deep-link goes to the Programme tab (where Marie edits per-module objectifs), NOT to the Fiche.

## acceptance

- `evaluatePreflight` in `src/lib/preflight/document-preflight.ts` adds: `prixConvenu IS NULL` → severity `block` for convention type, with `focusKey: 'prixConvenu'`, deep-link to Fiche
- `modules.every(m => !m.objectifs?.trim())` (no module has objectifs) → severity `warn` for convention + programme types, with `focusKey: 'programme-modules'`, deep-link to Programme tab
- `publicVise IS NULL` → severity `warn` for programme type, with `focusKey: 'publicVise'`, deep-link to Fiche
- `PreflightFormation` interface extended with `prixConvenu: string | null`, `publicVise: string | null`, and `modules: Array<{ objectifs: string | null }>`
- All preflight loaders (`+layout.server.ts` and any per-document loader) updated to select the new formation columns AND aggregate module objectifs
- Fiche fields (`prixConvenu`, `publicVise`) tagged with matching `data-preflight-target` attributes; Programme tab module list anchor tagged with `data-preflight-target="programme-modules"` for deep-link focus

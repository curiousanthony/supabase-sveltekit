Foundational schema work for the entire Finances redesign. The current single-row financing model on `formations` (`typeFinancement`, `montantAccorde`, `financementAccorde`) cannot express multi-source funding, partial grants, status lifecycle, or per-source dossier references. This ticket creates the new `formation_funding_sources` table + canonical enums and adds an optional `fundingSourceId` FK on `formation_invoices`, plus a non-destructive backfill from the legacy single-row data. The legacy columns stay for one release cycle.

This ticket corresponds to the `schema-funding-sources` task in the redesign plan. It was originally mapped to T-50 in the plan front-matter, but T-50 was assigned to the unrelated pgTAP test-infra ticket; T-63 takes its place as the canonical ticket for that task.

## acceptance

- New file `src/lib/db/schema/funding-sources.ts` exports `fundingSourceType` enum, `fundingSourceStatus` enum, and `formationFundingSources` table with all columns per the plan
- `src/lib/db/schema/formations.ts` `formationInvoices` gains nullable `fundingSourceId uuid` with FK → `formation_funding_sources(id) ON DELETE SET NULL`
- `src/lib/db/relations.ts` wires `formations` → `formationFundingSources` (one-to-many) and `formationInvoices` → `formationFundingSources` (many-to-one)
- `src/lib/db/schema/index.ts` re-exports from `funding-sources.ts`
- Drizzle-generated migration `supabase/migrations/{ts}_funding_sources.sql` creates the two enums, the table (with FK + index on `formation_id`), the invoice FK column, and runs an idempotent backfill that maps `type_financement` (`CPF`/`OPCO`/else) → canonical `funding_source_type`, copies `montant_accorde` to `granted_amount`, and sets status to `Accordé` when `financement_accorde` is true otherwise `Demandé`
- Backfill is idempotent (uses `NOT EXISTS` guard) — re-running the migration in CI/preview branches is a no-op
- Legacy columns `formations.type_financement`, `formations.montant_accorde`, `formations.financement_accorde` are NOT dropped in this ticket (one-release-cycle deprecation window)
- Down migration drops the FK column, the table, then both enums in correct order
- `bun run db:generate` produces the expected diff; preview push succeeds; Drizzle TS types build without errors

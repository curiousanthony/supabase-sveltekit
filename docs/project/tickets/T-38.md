**Scope:** For **every** quest whose CTA sends Marie to another formation tab, the app should behave like a light **onboarding flow**: after the CTA, land on the right tab and **guide** her to the **fields or buttons** she must use—focus, scroll into view, and (ideally) **one primary target at a time** until the quest’s action is done. Fiche is one important destination, not the only one. The experience is **per formation** (same patterns repeat each time Marie opens a formation’s Suivi, not a one-time app tour).

**Delivery:** Prefer a **quick win** first (reuse deep-links, scroll + focus + single obvious highlight on the first missing control, or section-level pulse), then iterate toward full **step-by-step** sequences driven by quest metadata. Optional **Phase 1 brainstorm** (ux-designer + product) if we need to lock coach UI, dismiss rules, and how this intersects Qualiopi “clear professional workflow” expectations—skip if quick win is agreed without it.

Complements **T-13** (pre-flight “Compléter →” fix paths) and **T-10** (Documents `?quest=`); this ticket is the **universal quest-CTA guidance** layer.

## acceptance
- **MVP / quick win:** At least one additional quest class beyond current Documents deep-link behaves as: CTA → correct tab → **focused or clearly highlighted** primary control (or section) relevant to that quest; user is not left on a tab with no visual cue what to do next
- **Target end state:** Any quest with a cross-tab CTA has a defined target (or ordered targets) so Marie is guided to the right **fields or buttons**; sequences are **data-driven** per quest where feasible, not copy-pasted one-offs
- Dismiss / exit does not trap focus; keyboard and screen-reader order remain usable
- Document the deep-link or query contract (extend existing patterns where possible) so new quests can register targets without rewriting tab pages

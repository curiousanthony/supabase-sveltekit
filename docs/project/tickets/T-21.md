When closing a deal and creating a formation, the deal's devis transfers to the formation and auto-completes the devis quest.

## acceptance

- Devis from deal carries over to the new formation; Devis quest auto-completes on formation creation from deal

## security note (carried from T-20)

The current Storage RLS policies (`formation_docs_workspace_*` in
`20260410100100_*.sql`) key off `(storage.foldername(name))[1]::uuid → formations.id`.
Pre-formation deal-stage docs have **no formation**, so any `deals/{dealId}/...`
upload is rejected by RLS — or worse, devs may use the service role / a different
bucket and bypass workspace scoping.

**Required for this ticket** (or a sibling shipped together): mirror the SQL
schema change into the storage policy. Add a deal-aware branch so a folder of
the form `deals/{dealId}/...` is allowed iff the caller is a member of
`deals.workspace_id`. Apply the same AND-not-OR principle T-20 used for
`formation_documents` policies (see
`docs/team-artifacts/security/2026-04-21-T-20-deal-link-rls.md`).

## 1. Programme tab — Link a Bibliothèque programme to a Formation

**Route**: Formation detail > Programme tab

Currently the "Choisir" button in the Programme tab just navigates the user to the `/bibliotheque/programmes` route but does nothing else — the user can browse programmes but cannot actually select and link one back to the Formation.

**What needs to happen:**

- The "Choisir" flow must let the user pick a Programme from the Bibliothèque and link it to the current Formation.
- Once a Programme is linked, the Formation's data should be **prefilled/updated** from the Programme's own data: modality, price, duration, and any other overlapping fields.
- The linked Programme's **Modules** should become visible inside the Formation's Programme tab, along with each Module's associated **Supports** (material/documents from the Bibliothèque) and **Questionnaires**.
- The user should be able to edit the programme data within the Formation context (it's a copy/override, not a live reference — or clarify this with the user).

**Why it matters:** This is the central feature that ties the Bibliothèque content to a live Formation. Without it, Formations are disconnected from their pedagogical content.

## 2. Modules fields alignment (Bibliothèque)

**Route**: `/bibliotheque/programmes` module editing

The Module entity in the Bibliothèque must have the following fields to match what the Programme tab of a Formation expects. This is **critical** — the Programme tab cannot work properly until Modules carry the right data.

**Required Module fields:**

- **Titre** (text)
- **Durée en heures** (number, decimal allowed — e.g. 1.5h, 3.25h)
- **Description** (textarea)
- **Objectifs pédagogiques** (textarea — listing learning outcomes)
- **Modalité d'évaluation** (text or select — how learners are evaluated)
- **Supports** (relation to Bibliothèque items — pedagogical materials/documents)
- **Questionnaires** (relation to Bibliothèque questionnaires)

Ensure the DB schema, Drizzle schema, forms, and UI all reflect these fields. When a Programme is linked to a Formation (item 1), these Module fields and their linked Supports/Questionnaires must flow through.

## 3. Quests CTAs audit — stop sending users to Suivi

**Route**: Formation detail > Suivi tab + all quest CTA buttons

Many quest CTAs currently lead the user to the "Suivi" route/tab. The problem: the user **cannot take action** on the Suivi tab — it's a read-only progress view. Sending the user there to "complete" a quest makes no sense because there is nothing to click or fill.

**What needs to happen:**

- Audit every quest and its CTA destination.
- Reroute CTAs to the actual tab/form/action where the user can fulfill the quest requirement (e.g., "Upload formateur documents" should go to the Formateurs tab or CRM, not Suivi).
- Ensure every quest CTA leads to a place where the user can **take a concrete action** that progresses the quest toward completion.

**What needs to happen:**

- **In the CRM** (`/contacts/formateurs`): add the ability to **upload documents** per Formateur to a Supabase Storage bucket and **preview** them.
